> 基于 HowToCook 开源菜谱的移动端应用
> 创建日期：2025-11-02
> 开发方式：AI-Assisted Development (Vibe Coding)

## 📋 项目概述

### 项目背景

[HowToCook](https://github.com/Anduin2017/HowToCook) 是一个程序员友好的开源菜谱项目，采用结构化的 Markdown 格式记录菜谱，目前拥有超过 300 个菜谱。本项目旨在基于这些优质内容，开发一个移动端应用（App + 小程序），提供更优秀的做菜体验。

### 核心创新点

**卡片式做菜模式** - 将做菜步骤以卡片形式呈现，用户可以左右滑动查看每一步，避免做菜时需要频繁滚动屏幕。这种交互方式特别适合厨房场景。

### 项目目标

#### 短期目标（MVP - 6-8 周）
- ✅ 完成小程序和 App 的基础框架
- ✅ 实现核心功能（浏览、搜索、卡片式做菜）
- ✅ 实现辅助功能（收藏、购物清单、计时器）
- ✅ 发布到微信小程序和应用商店

#### 长期目标（3-6 个月）
- 🎯 构建用户社区（评论、晒图）
- 🎯 添加智能推荐功能
- 🎯 营养统计与健康管理
- 🎯 多语言支持

## 🎯 功能规划

### MVP 核心功能（必须）

#### 1. 菜谱浏览模块
- **首页**
  - 分类导航（素菜、荤菜、主食、汤粥等）
  - 推荐菜谱（按难度/热度）
  - 搜索入口
- **菜谱列表**
  - 按分类展示
  - 筛选功能（难度、烹饪时间）
  - 排序（难度、时间、热度）
- **菜谱详情**
  - 完整菜谱信息展示
  - 食材清单
  - 步骤预览
  - 收藏/分享按钮

#### 2. 卡片式做菜模式 ⭐（核心创新）
- **交互方式**
  - 左右滑动切换步骤
  - 点击"完成"自动跳转下一步
  - 步骤进度指示器
- **辅助功能**
  - 语音播报当前步骤
  - 防息屏保持（做菜时屏幕常亮）
  - 步骤图片放大查看
- **智能提示**
  - 关键操作提示（如"油温"、"火候"）
  - 预计用时显示
  - Tips 提示框

#### 3. 收藏功能
- 收藏菜谱到个人中心
- 浏览历史记录
- "最近做过"标记

#### 4. 购物清单
- 根据菜谱自动生成食材清单
- 支持多个菜谱合并清单
- 勾选已购买项
- 清单分享功能

#### 5. 计时器
- 支持多个计时器同时运行
- 预设时间快捷选择
- 倒计时结束提醒

### 后期扩展功能（V2.0+）

#### 6. 用户系统
- 手机号/微信登录
- 个人资料管理
- 跨设备数据同步

#### 7. 社区互动
- 菜谱评论与打分
- 上传做菜成果图
- 个人做菜笔记（改良版本）
- 点赞与关注

#### 8. 智能推荐
- 根据时间推荐（早中晚餐）
- 根据食材推荐（我有什么食材）
- 根据难度推荐（新手友好）
- 随机推荐（解决选择困难）

#### 9. 营养统计
- 每日营养摄入统计
- 卡路里计算
- 健康饮食建议

#### 10. 数据统计
- 做菜记录日历
- 成就系统（徽章）
- 技能树（解锁新菜谱）

## 🛠️ 技术选型

### 前端框架
- **Taro 3** - 多端统一开发框架
- **React 18** - UI 框架
- **TypeScript** - 类型安全

### 选择理由
1. **一码多端**：Taro 支持编译到微信/支付宝小程序、H5、React Native
2. **React 生态**：你熟悉 React，开发效率高
3. **TypeScript**：类型安全，适合 AI 辅助开发
4. **成熟稳定**：Taro 3 已经非常成熟，社区活跃

### UI 组件库
- **Taro UI** - 官方组件库
- **NutUI (京东)** - 备选方案

### 状态管理
- **Zustand** - 轻量级，适合中小型项目
- **React Query** - 数据请求与缓存（后期）

### 本地存储
- **IndexedDB** (H5/App) - 结构化数据存储
- **Storage API** (小程序) - 键值对存储
- **SQLite** (App) - 可选，性能更好

### 工具链
- **Vite** - 构建工具
- **ESLint + Prettier** - 代码规范
- **Husky** - Git hooks
- **Commitlint** - 提交规范

## 📊 数据方案

### MVP 阶段：纯本地方案（方案 A）

#### 数据流
```
Markdown 菜谱 (原始数据)
    ↓ 脚本解析
JSON 数据 (标准化格式)
    ↓ 打包
App Assets (内置数据)
    ↓ 首次启动
IndexedDB / Storage (本地数据库)
```

#### 优势
- ✅ 零成本（无需服务器）
- ✅ 离线可用
- ✅ 响应速度快
- ✅ 开发周期短

#### 劣势
- ❌ 无法实时更新菜谱
- ❌ 无法支持用户系统
- ❌ 数据无法跨设备同步

### V2.0：云端 + 本地混合方案（方案 B）

#### 架构
```
云端 API (Node.js/Deno)
    ↓ CDN 加速
App 请求 + 缓存
    ↓ 本地存储
IndexedDB + Storage
```

#### 新增能力
- ✅ 实时更新菜谱
- ✅ 用户系统与数据同步
- ✅ 社区功能（评论、晒图）

## 🎨 设计原则

### 1. 简洁明了
- 界面简洁，突出内容
- 减少不必要的装饰元素
- 信息层级清晰

### 2. 操作便捷
- 一键开始做菜
- 最少步骤完成操作
- 手势操作流畅

### 3. 厨房友好
- 大按钮，易于点击
- 防误触设计
- 防油污屏幕也能操作

### 4. 程序员思维
- 保留原项目的结构化特点
- 精确的用量和时间
- 清晰的步骤顺序
